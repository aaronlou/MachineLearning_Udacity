问题 2 - 训练及测试¶

将数据集按一定比例分为训练用的数据集和测试用的数据集对学习算法有什么好处？

如果用模型已经见过的数据，例如部分训练集数据进行测试，又有什么坏处？

提示： 如果没有数据来对模型进行测试，会出现什么问题？

问题 2 - 回答:¶

  • 机器学习算法的核心思想在于从给定的样本集中寻找到特定的模式，然后将之作为此类样本总体的模式。对于从该总体产生的新样本，该模式应该同样适用，这便是泛化能力。如果我们用训练集来搭建模型，那么模型会针对这些训练集的数据的进行拟合，且进行一定最优化，这就带来过拟合的风险。用一个独立的测试集来检验模型在新样本上的预测能力，较客观地测量了模型的泛化能力。

  • 如果我们仍用模型已经见过的数据（从原训练集抽样部分）来测试，并不能客观地反映模型的泛化能力，因为模型在训练过程中就已经对这些样本进行了拟合。

编程练习3：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict的R2值，作为对其表现的评判。
  • 将他们的表现评分储存到score变量中。

或

  • (可选) 不使用任何外部库，参考决定系数的定义进行计算，这也可以帮助你更好的理解决定系数在什么情况下等于0或等于1。

from sklearn.metrics import r2_score
def performance_metric(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""

问题 8 - 回答：¶

  • 在构建模型时，我们需要区分独立的训练集与测试集，以保证在测试集上的所体现出来的评分客观地反映了模型在未见过的数据集上的泛化能力。K折交叉验证是说将可用数据集随机分为K等份，每次选择其中一份数据作为测试集，其余K-1份作为训练集，以此迭代K次，对于所得到的K个结果进行平均，即为最终模型的整体表现。

  • 在GridSearchCV的参数列表中，有一个cv的参数代表交叉验证的选项，默认为3折交叉验证。当我们进行了交叉验证后，最终的评分是在多次交叉验证上取的平均值，也更恰当地代表了模型的一般表现，而非在某次数据拆分情况所能达到的最优值。

  • cv_results包括了每个参数水平下训练集上模型训练的平均时间(k-fold拆分的平均值)、验证集上模型评价的平均时间；还包括每个参数水平上的模型,K次拆分条件下模型训练的各项平均表现值，例如训练集上的得分、验证集上的得分，同时也含有每一次拆分的训练集得分、验证集得分；还包括了K次拆分得分的标准差。基于这些信息，我们就可以判断在哪个参数水平上，模型的表现最优。
  • 网络搜索的目标在于选择模型，交叉验证的目标在于评价模型，有了交叉验证，我们最终用于评价模型的指标是K次训练结果的平均值，在此基础上选择的最优参数组合，不容易产生过拟合的现象。反之，如果没有交叉验证，那么我们在选择最优参数时，基于的仅是某一种数据集拆分，如果模型比较复杂从而产生了过拟合，那么这时的准确度评价是不客观的。

  5. 定义 'grid' 变量: 使用 sklearn.model_selection 中的 GridSearchCV 创建一个网格搜索对象；将变量'regressor', 'params', 'scoring_fnc'和 'cross_validator' 作为参数传至这个对象构造函数中；

from sklearn.model_selection import KFold
from sklearn.tree import DecisionTreeRegressor
from sklearn.model_selection import GridSearchCV
from sklearn.metrics import make_scorer 


def fit_model(X, y):
    """ 基于输入数据 [X,y]，利于网格搜索找到最优的决策树模型"""
    
    cross_validator = KFold(n_splits=3,shuffle=True,random_state=30).split(X)
    
    regressor = DecisionTreeRegressor()

    params = {'max_depth': list(xrange(1,11)) }

    scoring_func = make_scorer(performance_metric)

    grid = GridSearchCV(regressor,params,scoring_func,cv=cross_validator)

    # 基于输入数据 [X,y]，进行网格搜索
    grid = grid.fit(X, y)
    # print pd.DataFrame(grid.cv_results_)
    # 返回网格搜索后的最优模型
    return grid.best_estimator_

一个最优的模型不一定是一个健壮模型。有的时候模型会过于复杂或者过于简单，以致于难以泛化新增添的数据；有的时候模型采用的学习算法并不适用于特定的数据结构；有的时候样本本身可能有太多噪点或样本过少，使得模型无法准确地预测目标变量。这些情况下我们会说模型是欠拟合的。
